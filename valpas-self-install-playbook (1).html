Phase 01

Before the Meeting

Arrive ready. Hotel can start the day after the call.
#StepWhereOwnerNotes
01 Flagged as ready for kickoff during configuration stage Copper OPS Triggered when config checklist is complete: hardware shipped, account created, room list received.
02 Confirm hardware delivered to hotel Notion OPS Check shipping tracking. Do not schedule call until confirmed received.
03 Impersonate hotel account in dashboard — verify access is clean and ready Dashboard OPS Check: account active, correct permissions, no leftover rooms, gateway configured and paired.
04 Confirm room list is complete (room numbers + bed count per room) Notion OPS Collected during configuration phase via template. Verify bed types: legs vs platform/wall mount.
05 Send pre-call email: quick guide, app links, Calendly link Email OPS / CSM Use saved template. Attach quick guide. Introduce May and Salah as ongoing contacts.
06 Confirm hotel contact has downloaded the app before the call Email OPS / CSM If not confirmed, follow up 24h before the call.
07 Call my mom and cry
Phase 02

During the Meeting

Hotel contact leaves confident. No need to call for basic steps.
#TopicToolKey Points to Cover
01 Router setup App + Guide Placement: max one wall from rooms. Ethernet vs Wi-Fi. Status lights: green = good, red = restart first, then contact OPS.
02 App login and Installation Mode App Confirm they can log in live. Profile → Installation Mode → tap "+" → add room → name it. Walk through naming convention.
03 Smart leg installation (beds with legs) App + Guide Head end = smart legs. Foot = basic legs. Corner beds: smart leg faces the room. Pairing: remove cassette → pair → remove battery strip → reinsert → confirm blinking red.
04 Wall mount installation (platform beds only) App + Guide Only cover if hotel has beds without legs. Use leg as height guide before screwing mount. Verify cassette is easily removable before finishing.
05 Routing devices (only if rooms go offline) App Do NOT open unless rooms stay offline after 8h. One near router, others between router and problem rooms. Remove battery strip to activate. Wait up to 8h.
06 What "done" looks like Dashboard All rooms visible in app. All cassettes blinking red on install. Router green. Hotel confirms to OPS by message when complete.
07 Troubleshooting during install Live Call Cassette not blinking → remove and reinsert. Room offline → check open tasks first, then add routing device. Router red → restart, then contact OPS.
08 Escalation path — what to do if something is broken Live Call ⚠ To design — define what the hotel handles vs what goes to OPS
Phase 03

After the Meeting

Confirm completion, catch issues, close the loop.
#StepWhereOwnerNotes
01 Hotel confirms installation complete (all rooms done) Email / Message Hotel Prompt during the call: "Send us a message when the last room is done."
02 Verify all rooms online in dashboard Dashboard OPS Check: all rooms visible, all cassettes active and online, no offline sensors.
03 Send post-install confirmation email Email OPS / CSM ⚠ To build — no standard template yet
04 Confirm hotel staff understand Prevention tasks (purple dot) and Maintenance tasks (yellow dot) Email OPS / CSM Cover in post-install email or confirm it was in the quick guide they received.
05 Mark installation complete in Notion Notion OPS Update status, log any issues, confirm account is handed to CS team (May / Salah).
06 30-day check-in Email CSM ⚠ To define — standard for all members, or triggered by issues only?
